# Crescent City History Project Overview

This project is a reproducible scholarly history of Crescent City and Del
Norte County, California. It combines hand-authored manuscript sources,
curated local data, deterministic figures, citation validation, and the
shared template renderer into one auditable research artifact.

## Current Shape

| Area | Current structure |
|---|---|
| Manuscript | 58 analyzed Markdown source files: 46 topical chapters, 4 Part openers, abstract, introduction, timeline, methods, reproducibility, references, and two appendices |
| Bibliography | Curated BibTeX database in `manuscript/references.bib`; cited-key, reserve-key, and unused-entry counts are reported by each pipeline run and audited in `docs/claim_ledger.md` |
| Figures | 24 PNG outputs plus 24 SVG siblings generated from `src/_figures/`, with a deterministic source-key, provenance, long-description, and checksum manifest in `output/figures/figure_manifest.json` |
| Data | 21 CSV/JSON/YAML inputs in `data/`, with event/data provenance, figure provenance, and current-event refresh metadata tested |
| Checks | Prose quality, heading levels, section anchors, citation resolution, bibliography consistency, documentation drift, and current-event data shape |
| Main PDF | `projects/crescent_city/output/pdf/crescent_city_combined.pdf` after rendering |
| Release copy | `output/crescent_city/crescent_city_combined.pdf` after copy-out |

The most recent local pipeline report in `output/pipeline_report.json`
records 5/5 checks passed, 59,086 words, 370 unique citations, average
FKGL 17.40, citation density 6.26 per 1,000 words, and 24 generated
figures.

## Reproducibility Boundary

The project is designed to run offline after dependency installation.
Figures are deterministic matplotlib functions over checked-in data or
explicitly schematic geometry. Citation validation is performed against
the local BibTeX file. Current-event rows add `checked_as_of`,
`source_tier`, and `refresh_trigger` metadata so a green build does not
quietly stand in for source freshness.

Generated artifacts belong in `output/`; source truth stays in
`manuscript/`, `data/`, `src/`, `scripts/`, and checked-in docs.
Project-facing documentation (`README.md`, `AGENTS.md`, and `docs/`) is
kept in the test surface alongside folder-level README/AGENTS files so
public instructions, agent instructions, figure counts, and manuscript
organization stay aligned.

The tests prove structural reproducibility, citation resolution, data
shape, and deterministic rendering. They do not prove every historical
claim true. Volatile civic, scientific, and Indigenous-history claims
still require periodic source review through `docs/claim_ledger.md`,
`docs/source_to_claim_audit.md`, `docs/current_events_refresh.md`, and
`docs/sources_provenance_ethics.md`, while `docs/data_validation_qa.md`
separates implemented checks from planned checks. The claim ledger also
names the reserve bibliography keys that may remain uncited without
becoming silent reference drift.
